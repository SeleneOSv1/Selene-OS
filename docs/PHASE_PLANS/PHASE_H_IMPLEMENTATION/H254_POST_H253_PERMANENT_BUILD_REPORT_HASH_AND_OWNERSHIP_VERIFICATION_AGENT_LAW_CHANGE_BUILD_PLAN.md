## Agent-Law Change

H254 permanently refines repository agent law by requiring same-task repo verification before Codex may state:

- a landed build hash
- a pinned `HEAD == ...` value in a self-authored build instruction
- a prior or prerequisite build hash
- an exact symbol carrier
- an exact file or layer ownership claim

The required verification is now exactly:

- current landed commit hash must come from direct `git rev-parse HEAD`
- prior or prerequisite build hashes must come from direct `git rev-parse`, `git log`, or exact ancestor queries in the same task
- exact symbol or state ownership must come from direct `rg` or equivalent inspection against the exact named files in the same task
- when a symbol exists in both bridge and shell or across multiple layers, the split ownership must be reported exactly rather than collapsed into one carrier

The following are now permanently forbidden under repository agent law:

- reusing remembered full hashes from earlier assistant messages
- inferring exact ownership from summary memory when current repo truth can be queried directly
- collapsing bridge-owned canonical fields and shell-owned derived state into one carrier when repo truth shows a split
- stating an exact hash or ownership claim as definitive without same-task verification

If same-task verification cannot be completed, Codex must now report that claim as `unverified` and must not pin a self-authored build instruction or final report to that claim.
